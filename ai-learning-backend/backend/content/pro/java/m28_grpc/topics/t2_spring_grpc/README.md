## Key Concepts
- **`@GrpcService`**: Spring stereotype for gRPC service implementations (like `@RestController`)
- Extends `ServiceNameGrpc.ServiceNameImplBase` — generated by `protoc`
- **`responseObserver.onNext(response)`**: send response message (unary: exactly once)
- **`responseObserver.onCompleted()`**: MANDATORY — client hangs forever if omitted
- `onNext()` and `onCompleted()` vs `onError()`: mutually exclusive — never mix
- **`@GrpcClient("name")`**: injects blocking/async stub; name matches `grpc.client.name.*` in yml
- **`grpc.server.port: 9090`**: gRPC runs on 9090; HTTP stays on 8080 — both coexist
- **`grpc.client.NAME.address: static://host:port`**: client routing configuration
- **`negotiation-type: plaintext`**: dev only; production always TLS
- **`StatusRuntimeException`**: gRPC error — contains `Status.Code` (INVALID_ARGUMENT, NOT_FOUND, UNAVAILABLE, DEADLINE_EXCEEDED)
- `Status.INVALID_ARGUMENT.withDescription().asRuntimeException()`: error a call
- `@GrpcGlobalServerInterceptor`: auto-applied to ALL gRPC services (like Spring `HandlerInterceptor`)
- `Metadata.Key.of()`: defines gRPC header key (like HTTP header name)
- `Context.key()`: gRPC thread-local for passing interceptor values to service methods
- **Retry policy**: DEADLINE_EXCEEDED → retry once; UNAVAILABLE → don't retry
- **Round-robin LB**: `default-load-balancing-policy: round_robin` + multi-address
- gRPC health: `HealthCheckResponse.ServingStatus.SERVING` + Spring Actuator integration
