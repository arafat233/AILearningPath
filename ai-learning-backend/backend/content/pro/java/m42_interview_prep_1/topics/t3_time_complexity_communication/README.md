# Topic 3: Time Complexity Communication — Speaking Complexity Fluently

**Module**: M42 | **Difficulty**: ⭐⭐⭐⭐⭐

## Key Concepts
- Always say 'O(X) because [reason]' — never just 'O(n)'
- Common complexities and their causes: O(1) hashmap, O(log n) binary search, O(n) single pass, O(n²) nested loop
- Space complexity: count auxiliary structures, not input size
- Amortized analysis: ArrayList.add() is O(1) amortized despite occasional O(n) resize
- Best/worst/average: quicksort O(n log n) avg O(n²) worst — always mention when relevant
- Trade-off language: 'We can reduce time to O(X) at the cost of O(Y) space'
